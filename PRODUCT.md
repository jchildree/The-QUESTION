# Product

## Register

product

## Users

Software engineers and architects using Claude Code for investigation work:
debugging hard bugs, stress-testing designs, attribution-backed architectural
decisions. They distrust first-pass answers. CLI-native -- already in the
terminal when this plugin fires.

## Product Purpose

A Claude Code plugin enforcing investigative discipline: reproduce before
hypothesizing, attribute every finding, verify before fixing. Six skills
shipping a zero-hard-dependency workflow -- investigation board, provenance
tracking, compression modes -- built on the Vic Sage persona. Success: a root
cause confirmed green (verified) before a fix is applied, and a board that
persists findings across cases.

## Brand Personality

Cold, Precise, Relentless. No warmth, no filler -- only truth once you stop
accepting the first thing you're told. The persona is an absence, not a
costume: faceless because identity would compromise the investigation. Voice
is terse, skeptical, exact. Function over costume, always.

## Anti-references

- GitHub Copilot UI: AI-assistant friendliness, warm blues, reassuring copy,
  rounded softness. The opposite of what this does.
- Lazy first-pass answers: any pattern that accepts the first hypothesis.
- Obsidian-style glassmorphism / neon dark modes: aesthetic over precision.
- SaaS landing-page warmth: cream/sand backgrounds, gradient CTAs, card grids.
- Unattributed claims: anything presenting a finding without a re-runnable
  verify check.

## Design Principles

1. Function over costume. Persona, board, and provenance are enhancements,
   never hard dependencies. If a layer is unreachable, the investigation runs.
2. No green without verify. Confidence climbs by evidence; never hand-waved.
3. One finding at a time. Escalate to a follow-up only on genuine vagueness.
4. The archive never forgets. Closed cases persist; cross-case recall prevents
   repeat mistakes.
5. Trust nothing unattributed. Every claim carries source, method, verify.

## Accessibility & Inclusion

CLI-first context. No visual requirements for current surface. Future UI:
WCAG 2.1 AA for any rendered board or report view. High-contrast dark mode
as default, not optional.
